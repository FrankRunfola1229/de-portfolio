Keys, grain, and models that don’t rot.

Dim / Fact basics
Pick a grain, then design facts and dims around it.
-- Example grain: 1 row per (date, customer, product)
CREATE TABLE fact_sales (
  sales_date   DATE,
  customer_id  INT,
  product_id   INT,
  qty          INT,
  revenue      DECIMAL(18,2)
);

CREATE TABLE dim_customer (
  customer_id INT PRIMARY KEY,
  customer_name VARCHAR(200),
  segment VARCHAR(50)
);
3NF reminder
Normalize sources → then publish star schema for analytics.
-- 3NF idea: separate entities
CREATE TABLE customer (
  customer_id INT PRIMARY KEY,
  customer_name VARCHAR(200)
);

CREATE TABLE address (
  address_id INT PRIMARY KEY,
  customer_id INT,
  city VARCHAR(100),
  state VARCHAR(50)
);
